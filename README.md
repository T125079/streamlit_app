===============授業で扱っていない部品===============
1.st.page_link
目的: データの出典元（e-Stat）へ、直感的にアクセスできるリンクを提供するため。
使用箇所: 説明ページの「使用データ」セクション。

2.st.metric
目的: 調査年や有効回答数などの「統計の基本情報」を、大きな数字で強調して表示して、データの規模感を一目で伝えるため。
使用箇所: 説明ページの「基本情報」セクション。

4.st.session_state と st.stop
目的: アプリの「説明ページ」と「グラフ表示ページ」を切り替えるフラグを管理し、ユーザーに目的を正しく伝えるため。
使用箇所: アプリ全体の画面遷移動作。

3.st.toggle
目的: 「表を表示する」という設定を、クリックではなくスイッチ操作で切り替えられるようにして操作性を向上させるため。
使用箇所: サイドバーの「フィルターオプション」。

5.st.radio
役割: 複数の選択肢（グラフの種類など）から必ず1つを選ばせるため。
使用箇所: サイドバーのグラフ切り替え、および特定の年代、性別の選択。

6.st.expander
役割: 補足情報や詳細な設定項目を折りたたんで収納し、画面が情報過多になるのを防ぐため。
使用箇所: 「調査内容の詳細」や「表示項目オプション」。

7.st.divider()
役割: セクション間に区切り線を引き、視覚的に情報のまとまりを分かりやすくするため。
使用箇所: 説明ページの各セクション間。

8.st.info
役割: グラフの読み取り結果を、背景色付きのボックスで目立たせるため。
使用箇所: 各グラフの下部にある「データの読み取り」。

9.st.warning
役割: フィルター設定が存在しない場合の注意喚起を、背景色付きのボックスで目立たせるため。
使用箇所: フィルター設定が存在しない場合のエラーメッセージ。

10.st.markdown
役割: HTMLや太字を利用して、円グラフの凡例を中央寄せにするなど柔軟なレイアウトを実現するため。
使用箇所: 円グラフの凡例表示。
